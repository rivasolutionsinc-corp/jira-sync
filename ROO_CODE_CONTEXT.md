## 2. Context & Problem Analysis

### Current State (uvx)
```json
"atlassian": {
  "command": "uvx",
  "args": ["--from", "mcp-atlassian==0.11.9", "mcp-atlassian"],
  "env": { 6 secrets }
}
```

### Problems Identified
1. **Host Dependency:** Requires Python/uv toolchain on every machine
2. **Runtime Resolution:** Version pinning is fragile — `uvx` resolves at runtime
3. **No Isolation:** Runs in host environment, pollutes `~/.cache/uv`
4. **No Health Checks:** No restart policy or liveness probes
5. **Maintenance Burden:** Difficult to reproduce across team members

### Goal
Migrate to Docker Compose for reproducibility, isolation, and maintainability while preserving backward compatibility with `mcp_client.py` wrapper.

---

## 3. Architectural Decisions (ADR-001)

### Decision 1: Docker Compose over `docker run`
- Existing MCP servers use `docker run -i --rm` directly in `mcp_settings.json`
- `mcp-atlassian` requires 6 env vars + potential future SSL cert mounts
- `docker-compose.yml` provides declarative, version-controlled configuration
- Easier to extend with volumes, networks, or additional services

### Decision 2: Local Dockerfile Build
- `mcp-atlassian` does NOT publish official Docker image to GHCR/Docker Hub
- Canonical distribution is via PyPI (`pip install mcp-atlassian==0.11.9`)
- Build minimal `Dockerfile` on `python:3.12-slim` base image
- Ensures version parity with current `uvx` pin

### Decision 3: stdio Transport Preservation
- MCP protocol uses stdin/stdout for JSON-RPC communication
- `docker compose run --rm -T` preserves stdio (no TTY allocation)
- `-T` flag is critical: disables pseudo-TTY, prevents JSON-RPC framing corruption
- Maintains backward compatibility with Roo Code MCP framework

### Decision 4: Secret Isolation via `.env` File
- Tokens must NOT be embedded in `docker-compose.yml` (committed to git)
- Docker Compose auto-loads `.env` from working directory
- `.env` already in `.gitignore` — safe for local credentials
- Follows 12-factor app pattern for environment-based configuration

### Decision 5: Absolute Paths in mcp_settings.json
- `mcp_settings.json` is global config invoked from arbitrary working directories
- Relative paths would break when Roo Code runs from different directories
- Absolute paths to project's `docker-compose.yml` and `.env` are required

---

## 4. Implementation Blueprint

### File 1: `docker-compose.yml`
```yaml
version: '3.8'

services:
  mcp-atlassian:
    build:
      context: .
      dockerfile: Dockerfile
    stdin_open: true
    tty: false
    environment:
      - JIRA_URL=${JIRA_URL}
      - JIRA_USERNAME=${JIRA_USERNAME}
      - JIRA_API_TOKEN=${JIRA_API_TOKEN}
      - CONFLUENCE_URL=${CONFLUENCE_URL}
      - CONFLUENCE_USERNAME=${CONFLUENCE_USERNAME}
      - CONFLUENCE_API_TOKEN=${CONFLUENCE_API_TOKEN}
    restart: "no"
```

**Key Points:**
- `build:` section references local `Dockerfile`
- `stdin_open: true` + `tty: false` preserves stdio for MCP protocol
- `environment:` uses variable substitution from `.env`
- `restart: "no"` — Roo Code manages lifecycle

### File 2: `Dockerfile`
```dockerfile
FROM python:3.12-slim

RUN pip install --no-cache-dir mcp-atlassian==0.11.9

ENTRYPOINT ["mcp-atlassian"]
```

**Key Points:**
- `python:3.12-slim` — minimal base image (~150MB)
- `--no-cache-dir` — reduces image size
- Version pinned to `0.11.9` (matches current `uvx` pin)
- `ENTRYPOINT` ensures container runs `mcp-atlassian` directly

### File 3: `.env`
```dotenv
JIRA_URL=https://jira.example.local
JIRA_USERNAME=username@example.com
```
